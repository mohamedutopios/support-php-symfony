TP Openstack

Les opérations à réaliser :

  • Vous allez créer un nouveau groupe : GroupeDev.
  • Ce groupe aura les droits reader sur le projet que vous allez "Reader"
  • Vous allez créer 2 utilisateurs : Thomas et Marc.
  • Vous allez créer un projet : web-app.
  • Marc appartient au groupe GroupeDev. Il faudra vérifier que Marc dispose uniquement des droits auquel il a le droit pour ce projet.
  • Dans le projet web-app, vous allez créer les éléments suivants :
    • Vous allez créer un routeur "routeur-dev"
    • Vous allez créer 2 réseaux : reseau-public et un reseau-private. Vous choisirez les plages IP que vous jugerez pertinents pour les sous-réseaux (1 sous-reseau par réseau)
    • Vous aurez besoin d'une image ubuntu 18.04 et une 22.04.
    • Un Gabarit pour les 2 vm -> cpu : 2, ram : 2048 et 10GO.
    • Vous aurez besoin de 2 VM : vm1 avec l'image 18.04 et vm2 22.04.
    • Chaque VM aura son propre groupe de sécurité.
    • 1 seule IP flottante sera nécessaire pour l'ensemble du projet.
    • Vous aller déployer le projet suivant : La partie front-end dans la vm1 du reseau-public : https://github.com/mohamedutopios/web-blog.git dans un serveur nginx (avec ou sans container).
    • Vous aller déployer l'autre projet : La partie back-end dans la vm2 du reseau-private : https://github.com/mohamedutopios/api-express-posts.git directement dans le serveur (avec ou sans container).
    • Sur la vm2, vous aller essayer d'installer un certain nombe d'éléments pendant phase de création de la vm grâce
    • Les 2 projets doivent communiquer entre eux pour faire fonctionner l'application.
    • Vous vous connecterez sur votre machine hôte sur le bon port pour afficher la partie front-end du projet.
